# Alembic Migration Versions

Generated migration files. Each file represents one schema change, applied in order by Alembic.

## Filename convention

`YYYYMMDD_HHMM_<revision>_<slug>.py`

The timestamp prefix gives lexical ordering. The revision is a short hex identifier. The slug is a human-readable summary.

## Discipline

- Never edit a migration after it has been applied to any environment beyond your local dev database.
- Never manually delete a migration file from this folder. To revert a schema change, write a new migration that reverses it.
- Always review autogenerated migrations before applying them. Alembic is good but not perfect.
